## hearthmirror-rs 集成路线图（Plan ↦ 3 changes 拆分）

> 来源：[`docs/superpowers/plans/2026-04-20-integrate-hearthmirror-rs.md`](../../docs/superpowers/plans/2026-04-20-integrate-hearthmirror-rs.md) §1-§8。
> 该 plan 已拆为下述 3 个 OpenSpec changes，按依赖顺序执行：

### ✓✓ 5d. `verify-hearthmirror-on-real-hs`（spike，已实施）

**类型**：spike / 数据收集。依赖：`add-hearthmirror-reflection-methods`（已完成）。

**结果**：✅ 高价值产出。1-2h 投入，抓到 4 个对后续工作有方向性影响的事实：

- **F-1 (Critical, P0 blocking)**：`MonoRuntime::init()` 在真实炉石上以 `STATUS_ACCESS_VIOLATION` 崩溃。Root cause 已定位到 `runtime.rs:97` 的 `mono.size.min(0x100_000)` 1MB PE 读取上限把 `pelite::PeView::module` 喂坏 — `exports.by().name()` 跟随 RVA 越界读触发 Windows SEH。**12 个反射方法全部 blocked**。修复见 5d-fix。
- **F-2 (Positive)**：去 cap 后 init 链 6 步全绿（process / handle / module / PE / disasm pattern A / root domain `0x0B442E70`）— 现有架构无更深层错误。
- **F-3 (Informational)**：mono dll 实际位置 `MonoBleedingEdge/EmbedRuntime/mono-2.0-bdwgc.dll`，非 `Rewrite_Design.md` 假设的 `Mono/`。
- **F-4 (Informational, ⚠️ critical implication for 5e)**：**生产炉石 Unity 版本 = 2022.3.62f2**，不是计划中的 2021.3。

参考：[`openspec/changes/verify-hearthmirror-on-real-hs/`](verify-hearthmirror-on-real-hs/) + [`docs/spikes/0003-*.md`](../../docs/spikes/0003-hearthmirror-reflection-runtime-validation.md)

### ✓✓ 5d-fix. `fix-hearthmirror-pe-read-cap`（已实施 + Run 2 验证完成）

**类型**：bugfix。依赖：无。**5e/5f 的硬阻塞** — 不修，所有真机验证全部失败。

**范围**：把 `runtime.rs:97` 的 `let pe_size = mono.size.min(0x100_000) as usize;` 改为 `let pe_size = mono.size as usize;`（一行 fix），加 1 个回归集成测试。源于 spike 0003 R-1。

**结果**：✅ fix 已提交（`120d33e`），回归测试 `integration_runtime_init.rs` 已添加。**Run 2 已采集**（spike 0003）：
- **F-5 (Positive)**：init 不再 ACCESS_VIOLATION，12 方法全部能进入入口（< 1ms each）。fix 验收完成。
- **F-6 (Critical, defer to 5e/5f)**：12/12 方法因 `probe_class_def_table_offset` 启发式在 Unity 2022.3 失效而 fail —— **正好印证 5e/5f 设计动机**，**不是字段名飘移**。
- **F-7 (Minor, optional)**：`probe.rs:35-36` 占位错误信息 `<probe>.<probed>` 无诊断价值，可独立 5-10 分钟微 fix。
- **5e baseline 决策修订**：用 `unity-2021.3.json` 起步 + 让 `OffsetProber` 在运行时refine — 不必前置投入手工构造 `unity-2022.3.json`。

参考：[`openspec/changes/fix-hearthmirror-pe-read-cap/`](fix-hearthmirror-pe-read-cap/) + [`docs/spikes/0003-*.md ## Run 2`](../../docs/spikes/0003-hearthmirror-reflection-runtime-validation.md)

### ✓✓ 5d-fix-2. `fix-hearthmirror-probe-error-msg`（已实施 — F-7 诊断改善）

**类型**：诊断 hygiene。依赖：无。**5e 的友军** — 让任何 probe 失败的错误信息一眼能定位 site，而不是统一显示 `<probe>.<probed>`。

**范围**：`probe_field_offset` 加 2 个 `&str` 参数（`owner_class`, `owner_field`），唯一 caller (`runtime.rs:159` 的 `MonoDomain.loaded_images` 探测) 同步更新。源于 spike 0003 F-7。

**结果**：✅ 实施完成（`115cf0a`）。`cargo test --lib` 33/0/1，clippy 0 错。**端到端验证证据**：5e 启动前的 `discover_domain_offsets` 测试现在 fail 时显示 `FieldNotFound { class: "MonoDomain", field: "loaded_images" }`，而不是过去的 `<probe>.<probed>`。Spike 0003 Run 2 F-6 的 root-cause 分析同步修订（实际失败 site = `discover_offsets` → `probe_field_offset`，**不是** `probe_class_def_table_offset` — 进一步把 5e 顶到关键路径）。

参考：[`openspec/changes/fix-hearthmirror-probe-error-msg/`](fix-hearthmirror-probe-error-msg/)

### ✓✓ 5e. `add-hearthmirror-offset-probing`（已实施 + Run 3 验证完成）

**类型**：稳定性 / 基础设施。依赖：`fix-hearthmirror-pe-read-cap`（强阻塞）+ `verify-hearthmirror-on-real-hs` Run 2 数据。

**范围**：移植 hearthmirror-rs 的 `iced-x86` 反汇编引擎 + `MonoOffsets` JSON 配置 + `OffsetProber`，替换 `field_paths.rs` 中硬编码的 13 个 Mono 结构偏移与 `runtime.rs` 中两个手写 byte-pattern 探测。业务字段名常量与 12 个反射方法链路保持不变。兑现 ADR 0001 约束 #5。

**结果**：✅ 全部交付。Phases 1–7 + 5.5 路由统一 + 6.5 Phase 6 Audit (D13 range-gate) 全部完成。Run 3 真机验证关键产出：

- **F-8 (fixed)**：`MonoAssembly.image` 应在 `+0x48` 而非 `+0x40`（MSVC 对 `MonoAssemblyName.public_key_token[17] + arch` 的 padding 把整个 inline `MonoAssemblyName` 拉到 0x40 字节）。`unity-2021.3.json` 已修正，`MonoImage` 4 个字符串槽（`+0x14/+0x18/+0x1C/+0x20`）的语义全部经 `diag_image` empirical 标注。
- **F-9 (informational)**：D13 range-gate 是 load-bearing — `mono_image_get_name` 反汇编一致返回 `0x1C`（短 asm 名，非 reflection 期望的 `0x14` 全路径），唯有 `sane_range=0x10..=0x18` 把 caller 钉在正确槽上。
- **F-10 (informational)**：4 个 profiled-thunk probe 一致返回 `0xE10` 这种 garbage displacement，range-gate fallback baseline 是正确响应。
- **F-11 (open, → 5f)**：`find_class` 仍假设 `MonoImage` 含一段 RID-indexed 的 flat `MonoClass*[]`（`probe_class_def_table_offset` 扫前 0x200 字节），但 Mono 实际通过 `MonoImage.class_cache` 的 `MonoInternalHashTable<MonoClass*>` (`+0x35C`) 走 token 哈希。这是 `dump_reflection` 11/12 仍然失败的唯一剩余障碍 → `add-hearthmirror-image-walking` (5f) 范围内。
- **acceptance**：`dump_reflection` 不再返回 12/12 的 `0x15` 错误；`getBattlegroundRatingInfo` 返回 `null`（class 未加载，主菜单态预期）；其余 11/12 现在挂在 `class_def_table` error 上，向 5f 移交。

参考：[`openspec/changes/add-hearthmirror-offset-probing/`](add-hearthmirror-offset-probing/) + [`docs/spikes/0003-*.md ## Run 3`](../../docs/spikes/0003-hearthmirror-reflection-runtime-validation.md)

### ✓✓ 5f. `add-hearthmirror-image-walking`（已实施完成）

**类型**：稳定性 / 内部重构。依赖：`add-hearthmirror-offset-probing`（必须先完成 — 用 `image.class_cache` / `class.parent` 等偏移）。

**范围**：移植 hearthmirror-rs 的 `MonoImage` (class_cache hashtable walking) + `MonoFieldDef` 独立模块 + `MonoClass.fields_recursive`（含继承链）。`MonoRuntime::find_class` 内部从 token 反向探测重构为 hashtable walking；删除 `probe_class_def_table_offset`。napi 公共 API 与 12 个反射方法链路不变。

**结果**：✅ 全部交付。Phases 1–7 全部完成（commit `b28ad63`）+ 3 个真机 integration 测试（`integration_image_walking.rs`）。关键实现选择：

- **F-11 (fixed)**：`MonoImage::enumerate_classes` 走 `image.class_cache = 0x35C` 处 **embedded** `MonoInternalHashTable`（不是指针 — JSON baseline `$class_cache_note` 是 source of truth）；遍历 `size` 个 bucket head，每条链用 `class.next_class_cache = 0xA0` 串起来。`size==0 → Ok(vec![])` + warn；`size>0` 但 0 valid class → `Err(ClassCacheEmpty)` 防御性告警。
- **新 API**：`MonoFieldDef { name, offset, type_ptr, is_static }`；`MonoClassRef::{parent, fields_recursive, find_field}` 带 32 层 hierarchy depth cap；`MonoObject::find_field` 经 vtable → class → `find_field` delegation；`MonoImage<'r>` view type。
- **删除**：`probe_class_def_table_offset` (~70 行) + `find_class_def_table_offset_cached` (~15 行) + `RuntimeCache::class_def_table_offset`；`ClassNotFound { name }` → `{ namespace, name }`。
- **测试**：56 baseline → 71 lib（+4 field + 6 class + 3 image + 2 ClassNotFound display）+ 3 新 integration (`integration_image_walking.rs`)；clippy `-D warnings` 0 错；grep `class_def_table` 仅剩 2 条 doc-comment 引用（记录"替换了哪条老路径"，spec Scenario 允许）。

参考：[`openspec/changes/add-hearthmirror-image-walking/`](add-hearthmirror-image-walking/)
