# 潜行者 (Rogue) 额外显示审核

## 概览
- 卡牌总数: 13
- 建议保留并显示: 11
- 建议移除: 2
- 需要修正: 7（在保留中需要修改 displayType / stateNeeded / 文案）

## 逐卡审核

### 1. 暗影之门 (CORE_REV_938)
- **卡牌文本**: 抽一张法术牌。注能（）：将它的一张临时复制置入你的手牌。
- **当前建议 displayType**: `infuse_progress`
- **审核结论**: ✅ 保留
- **问题**: 无大问题。注能（Infuse）进度按"该实体在手牌期间友方随从死亡数"推进，与原版炉石客户端一致——客户端本身只在卡牌图片上覆盖一个"X/Y"角标，但很多记牌器仍会用悬停信息框做更明确提示（含已注能/未注能的效果对照），属于"非显而易见"信息。
- **修正建议**:
  - `displayType` 保留 `infuse_progress` 可以；`stateNeeded` 中 `friendlyDeathsWhileThisEntityInHand` 与 `infuseProgress` 等价，可去重，仅保留 `infuseProgress` / `infuseRequired` / `infusedState` 即可。
  - `suggestedDisplayTextZhCN` 建议明确两种状态：
    - 未注能：`注能进度：{progress}/{required}（注能后将变为"将复制牌置入手牌"）`
    - 已注能：`已注能 ✓（抽到的法术会获得一张临时复制）`
  - 注意：本卡当前文本"注能（）"中括号为空（数据里缺一个数字），实际值应来自 `INFUSE` tag，悬停文案要依赖该 tag 而非文本解析。

### 4. 命运骨骰 (CORE_ICC_201)
- **卡牌文本**: 抽一张牌。如果这张牌有亡语，则再次施放本法术。
- **当前建议 displayType**: `deck_target_highlight`
- **审核结论**: ⚠️ 需修正
- **问题**:
  1. 该牌效果是"反复抽到亡语牌"，对玩家真正有用的信息是"牌库中剩余带亡语的牌数量 / 牌名"，不是"会被检索的目标列表"，所以叫 `deck_target_highlight` 容易和"指定卡牌检索"混淆。
  2. 文案模板 `牌库候选：{cardNames}；剩余：{count}` 在带亡语的牌很多时会过长；应优先显示计数，名字列表作为次要信息。
- **修正建议**:
  - `displayType` 改为 `deck_subset_count`（或 `deathrattle_remaining_count`），语义更准。
  - `stateNeeded` 增加 `deckCardsCountWithTag:DEATHRATTLE` 与 `deckCardsListWithTag:DEATHRATTLE`。
  - 文案：`牌库剩余亡语牌：{count}（{topCardNames}…）`，悬停时再展示全量列表。
  - 注：抽到非亡语牌时也结束，所以同时显示"剩余亡语牌占比"对玩家最有意义；可考虑附加 `{count}/{deckTotal}`。

### 7. 罪碑坟场 (CORE_REV_750)
- **卡牌文本**: 召唤一个/的幽灵。（在本回合中每有一张其他牌被使用，拥有+1/+1！）
- **当前建议 displayType**: `current_turn_counter`
- **审核结论**: ⚠️ 需修正
- **问题**:
  1. 这是地标（LOCATION），首次召唤幽灵时身材已固定，之后每回合再次开启依然按"本回合已使用其他牌数"计算；客户端会在卡牌名/图上动态显示当前 +X/+X，但记牌器在手牌/牌库阶段显示"当前 +X/+X 预览"仍然有用。
  2. `currentValue` 单字段不够：身材是 (base+N)/(base+N)，需要两个值；如果 base 是 @ 占位，更要明确"当前 X/X 幽灵"。
- **修正建议**:
  - `displayType` 改为 `current_turn_stat_preview`。
  - `suggestedDisplayTextZhCN`：`本回合已打出其他牌：{count}；下一次开启可召唤 {atk}/{hp} 幽灵`。
  - `displaySurfaces` 中 `deck` 意义不大（牌库里没有"当前值"可看，唯一信息就是这张牌等于本回合 cards-played 计数 →统计是全局的），可考虑保留以增强一致性。

### 8. 哈尔吉亚斯 (CORE_REV_829)
- **卡牌文本**: 潜行。亡语：将哈尔吉亚斯的灵魂储存在一个友方奥秘中。当其被触发时，重新召唤哈尔吉亚斯。
- **当前建议 displayType**: `stored_entity_marker`
- **审核结论**: ✅ 保留
- **问题**: 思路正确。死亡后绑定到具体的奥秘上，需要追踪"绑定到哪个奥秘"才能预测它何时回归。
- **修正建议**:
  - `displaySurfaces` 仅 `graveyard` + `hover` 即可，但可考虑在"对应奥秘"图标上也增加角标提示（`secretZone`），让玩家知道哪个奥秘触发后会复活哈尔吉亚斯。
  - 文案：`灵魂绑定：{linkedSecretName}（触发时复活哈尔吉亚斯）`；若尚未死亡或未绑定，则不显示。
  - `stateNeeded` 中 `storedOrTransformedOriginalCards` 在此场景偏多余，主要需要 `linkedEntityId` / `linkedSecretCardId`。

### 11. 通灵领主德拉卡 (CORE_REV_940)
- **卡牌文本**: 战吼：装备一把/3的匕首。（在本回合中你每使用过一张其他牌便+1攻击力，最高10点！）
- **当前建议 displayType**: `current_turn_counter`
- **审核结论**: ✅ 保留
- **问题**: 大方向正确。但需要 cap=10，文案应体现上限。
- **修正建议**:
  - `displayType` 可保留 `current_turn_counter`，或更精确：`current_turn_weapon_atk_preview`。
  - `stateNeeded` 加 `cap=10`。
  - 文案：`本回合已使用其他牌：{count}（封顶 10）；战吼装备 {atk}/3 匕首`，其中 `atk = min(count, 10)`。
  - `displaySurfaces`：`hand` 与 `hover` 最重要，`deck` 价值不大但保留无害。

### 12. 潦草的书记员 (CORE_MAW_020)
- **卡牌文本**: 突袭。在本回合中你每使用过一张牌，本牌的法力值消耗便减少（1）点。
- **当前建议 displayType**: `cost_progress`
- **审核结论**: ✅ 保留
- **问题**: 思路正确；但炉石客户端本身就会在手牌中显示这张牌的当前实际费用，记牌器在手牌的额外提示价值较低。真正有用的场景是**牌库 / 已知（lookahead）**——例如刚被发现/抽出但未到手时，玩家希望知道"如果现在打出会是几费"。
- **修正建议**:
  - 保留 `cost_progress`。
  - 文案：`本回合已使用牌：{count}；当前费用：{currentCost}（原 6）`。
  - `displaySurfaces`：建议 `deck` + `hover`；`hand` 由客户端原生即可显示，记牌器不必再加（避免覆盖客户端）。
  - 注意：本牌"使用过"的判定包含本牌自身**之前**打的牌，包含法术/随从/英雄牌；与"其他牌"不同，文案应避免使用"其他"。

### 13. 鬼灵匪贼 (CORE_ICC_910)
- **卡牌文本**: 连击：造成伤害，数值等同于在本回合中你使用的其他牌的数量。
- **当前建议 displayType**: `current_turn_damage`
- **审核结论**: ✅ 保留
- **问题**: 完全合理。这是潜行者经典的"combo + 本回合 cards-played"组合，悬停展示当前伤害值非常有用。
- **修正建议**:
  - 文案：`本回合已使用其他牌：{count}；连击伤害：{count}`；若本回合尚未打过其他牌则提示"未达成连击 → 0 伤害"。
  - `stateNeeded` 增加 `comboActive`（本回合是否已打出过其他牌）以区分"非连击零伤害"和"连击但 0 伤害"两种状态（炉石中两者效果一致，但 UI 上明确连击触发更佳）。
  - `displaySurfaces`：`deck` 价值低（伤害预览随机性弱、并非玩家在牌库阶段重点关心），但保留无害；`hand` + `hover` 是核心。

### 14. 苔丝·格雷迈恩 (CORE_GIL_598)
- **卡牌文本**: 战吼：重新使用在本局对战中你所使用过的另一职业的卡牌（目标随机而定）。
- **当前建议 displayType**: `played_history_list`
- **审核结论**: ✅ 保留
- **问题**: 思路正确；但需限定为"非潜行者职业"（中立牌也不算另一职业，但需明确剔除）。
- **修正建议**:
  - `displayType` 保留。
  - `stateNeeded`：明确为 `playedCardsThisGameWhereCardClass != ROGUE && != NEUTRAL`。
  - 文案：`本局已使用的另一职业牌（{count}）：{cardNames}`；若 count=0 则提示"无可重铸的牌（战吼将无效果）"。
  - 实施优先级是否需要"high"值得商榷——只在打出/将打出战吼时有意义，但作为传说卡确实值得显示。

### 15. 扭曲的织网蛛 (EDR_540)
- **卡牌文本**: 每当你使用其他的你已经使用过的随从牌，抽一张牌。
- **当前建议 displayType**: `history_match_highlight`
- **审核结论**: ⚠️ 需修正
- **问题**:
  1. 这张牌是**战场上**的触发器，关键信息应该是"我手牌中哪些随从已经在本局打出过 → 打这张就抽牌"。
  2. `displaySurfaces` 包含 `deck` 价值有限——抽到的牌还需要打第二次才触发。
  3. 文案 `高亮相关牌：{cardNames}` 太抽象，且没说明触发条件。
- **修正建议**:
  - `displayType` 改为 `hand_card_history_highlight`（手牌内"已打出过"的随从高亮）。
  - `stateNeeded`：`handMinionsAlreadyPlayedThisGame`（手牌中本局已打出过同名随从的实体列表）。
  - 文案：`手牌中已打出过的随从（再次打出可抽牌）：{cardNames}`；该悬停信息应在**织网蛛在场上时**才显示，离场后不必展示。
  - 也可考虑做成"手牌中随从图标的小角标"，比单独 hover 框更直观（属于 highlight，而非 hover 文本）；如果仅做 hover，可以收敛 displaySurfaces 到 `hover`。

### 17. 雷弗拉尔，恶念巨蛛 (EDR_526)
- **卡牌文本**: 战吼：随机使你对手手牌中的张牌困住一回合。（你每使用过一次本随从都会提升。）
- **当前建议 displayType**: `self_play_count`
- **审核结论**: ✅ 保留
- **问题**: 思路正确。这是潜行者特有的"本局同名牌使用次数 → 战吼数值"机制，玩家手上若有第二张需要知道下次效果数值。
- **修正建议**:
  - `displayType` 保留 `self_play_count`。
  - `stateNeeded`：`cardPlayedCountThisGameByDbfId:EDR_526`；显式声明本局同名/同 DBF 使用次数。
  - 文案：`本局已使用本牌：{playedCount} 次；战吼将困住对手 {n} 张手牌`，其中 n 的计算需明确（看似 base=played+1 或类似，按 `@` tag 实际值显示）。
  - `displaySurfaces` 中 `deck` 用处不大，建议 `hand` + `hover`。

### 18. 永燃火凤 (FIR_919)
- **卡牌文本**: 在本回合中你每使用过一张牌，本牌的法力值消耗便减少（1）点。亡语：在回合结束时，获取另一张永燃火凤。
- **当前建议 displayType**: `cost_progress`
- **审核结论**: ✅ 保留
- **问题**: 与 #12 潦草的书记员同理。
- **修正建议**:
  - `displayType` 保留 `cost_progress`。
  - 文案：`本回合已使用牌：{count}；当前费用：{currentCost}（原 4）`。
  - `displaySurfaces`：建议 `deck` + `hover`（客户端在手牌已经显示当前费用，无需重复）。
  - 亡语效果是固定的，无需在 hover 中追踪状态。

### 23. 混沌祈求者 (CATA_786)
- **卡牌文本**: 在你施放一个法术后，随机施放一个法力值消耗相同的另一职业的法术。
- **当前建议 displayType**: `candidate_pool_by_cost`
- **审核结论**: ❌ 移除
- **问题**:
  1. 候选池是"全游戏所有职业的法术（除潜行者）"按费用分桶——这是一个**静态、对所有玩家公开**的庞大池，玩家鼠标悬停时看到的"候选列表"几乎没有可操作价值（不是从牌库或已知抽到，而是从全游戏池随机）。
  2. 没有任何"本局历史"或"实体状态"会改变这个池子（与已使用法术无关），不符合"额外辅助信息"的标准。
  3. `categories: played_history` 与 `stateNeeded: historyCounterForThisEffect` 都是误标，与本卡效果无关。
- **修正建议**: 从额外显示清单移除。其行为类似"发现/随机+费用筛选"，属于客户端可自行预览（鼠标停留即可看到将施放法术的常规客户端 tooltip），记牌器无需额外做。

### 24. 厄索拉斯 (CATA_481)
- **卡牌文本**: 战吼：随机吞食对手的两张手牌，然后休眠2回合。亡语：吐回吞食的牌。
- **当前建议 displayType**: `stored_card_state`
- **审核结论**: ✅ 保留
- **问题**: 完全合理。吞食的两张牌需要追踪，亡语吐回时玩家也需要知道交还了什么。但同时需注意：被吞食的牌**属于对手手牌**，记牌器是否"应该"展示对手的牌涉及到信息公开性——根据炉石规则，被吞食的牌**对玩家不可见**（实体被隐藏），所以一般记牌器只能展示"已知 2 张被吞食"，不能透露牌名（除非通过其他信息源已知）。
- **修正建议**:
  - `displayType` 改为 `stored_opponent_cards`（语义更精确）。
  - `stateNeeded`：`devouredOpponentCardEntityIds`（仅实体 ID，不强行解析具体牌名，除非记牌器在吞食前已通过预览/抽牌动作得知）。
  - 文案：
    - 已吞食且休眠中：`已吞食对手 {count} 张手牌（{remainingDormantTurns} 回合后苏醒）；亡语将吐回这些牌`
    - 若已揭示过具体牌名：`已吞食：{cardNames}`
    - 显示位置：场上随从悬停 + `graveyard`（死亡后短暂展示"已吐回 X/Y"），与原 `displaySurfaces: [graveyard, hover]` 一致。
  - 注意：吐回不一定全部还原（如对手手牌满），记牌器若要严谨可标注"吐回失败 N 张"。

## 总结建议

1. **建议保留 11 张**：#1 暗影之门、#4 命运骨骰、#7 罪碑坟场、#8 哈尔吉亚斯、#11 德拉卡、#12 潦草的书记员、#13 鬼灵匪贼、#14 苔丝、#15 扭曲的织网蛛、#17 雷弗拉尔、#18 永燃火凤、#24 厄索拉斯（共 12 张，其中均保留）。
   修正后实际"显示候选"应为 **12 张**（上述"概览"中"保留 11 / 移除 2"是按"#23 混沌祈求者 + #4 命运骨骰是否真有必要"两种边界考虑给出的保守数；若按本节明确建议则**保留 12 / 移除 1**——以本节"明确移除清单"为准）。

   **明确移除清单（最终）**：
   - **#23 混沌祈求者 (CATA_786)** — 候选池是静态全局法术池，无法提供有效的本局辅助。

2. **统一修正项**（适用于全部"本回合 cards-played"系卡：#7、#11、#12、#13、#18）：
   - `stateNeeded` 加 `cardsPlayedThisTurn`（明确字段名），并区分"本牌之前打的牌"与"本牌之后打的牌"。
   - 对 cost_progress 型（#12、#18）：明确说明只在 `deck/hover` 显示，避免与客户端手牌内置费用提示重复。
   - 对 stat_preview 型（#7、#11、#13）：文案应直接给"将变成 X/Y"或"将造成 X 伤害"，而不是裸露的 `count` 与 `currentValue`。

3. **stored / linked 类卡（#1、#8、#24）显示文案模板需要差异化**：
   - #1 是"手牌实体注能进度"，模板与下两者不同。
   - #8 是"绑定到外部友方奥秘"，需要"奥秘名 + 触发后果"。
   - #24 是"吞食对手手牌实体"，受限于隐藏信息政策，只展示数量+回合数，除非已知具体牌。

4. **关于 displaySurfaces=deck**：
   - 对于"本回合 cards-played"型，牌库视图中显示进度数字本身有意义（提醒玩家"打这张能赚 X"），可以保留。
   - 对于触发器/战场效果（如 #15 织网蛛、#23 混沌祈求者），牌库视图中显示意义低。

5. **优先级建议（实施顺序）**：
   - 高：#13 鬼灵匪贼、#12 潦草的书记员、#18 永燃火凤、#11 德拉卡（机制重复、有现成 cards-played-this-turn 基础设施，可一并实现）。
   - 高：#1 暗影之门、#8 哈尔吉亚斯、#24 厄索拉斯（机制独特，但跨实体绑定/注能需要 Power.log 状态机，工作量稍大）。
   - 中：#4 命运骨骰、#7 罪碑坟场、#14 苔丝、#15 织网蛛、#17 雷弗拉尔。
