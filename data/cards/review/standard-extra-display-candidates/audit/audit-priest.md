# 牧师 (Priest) 额外显示审核

## 概览
- 卡牌总数: 13
- 建议保留并显示: 10
- 建议移除: 2
- 需要修正: 6（保留中需调整 displayType / stateNeeded / 文案的卡牌）

> 说明：本审核独立于 JSON 中已有 `extraDisplay` 字段。判断依据为效果是否真正依赖"无法从当前可见场面直接看出的对局历史信息"。固定可见效果、或者悬停标准卡图已能让玩家自己心算清楚的，建议移除。

## 逐卡审核

### 3. 永恒奴役 (CORE_ICC_213)
- **卡牌文本**: 发现一个在本局对战中死亡的友方随从，并召唤该随从。
- **当前建议 displayType**: `graveyard_pool`
- **审核结论**: ✅ 保留
- **问题**:
  - `stateNeeded` 写成了 `eligibleDeathrattleOrMinionPool`，但此牌并不限定亡语，叙述不准确；
  - `displaySurfaces` 含 `graveyard`，但发现行为发生在出牌瞬间，玩家最需要的是"打出前/手牌悬停"时预览，`graveyard` 面板可以保留但优先级低。
- **修正建议**:
  - `stateNeeded` 改为 `friendlyDeadMinionPoolThisGame`（去掉 deathrattle 字样）；
  - 悬停文案建议改为："发现候选（本局已死亡的友方随从，去重）：{cardNames}；共 {distinctCount} 种"。
  - 注意需要"去重"，因为发现是按"种类"挑而不是按"实例数"；同一张牌死多次只算一种候选。

### 5. "太阳裂片"莱拉 (CORE_UNG_963)
- **卡牌文本**: 每当你施放一个法术，随机将一张牧师法术牌置入你的手牌。
- **当前建议 displayType**: `candidate_pool`
- **审核结论**: ❌ 移除
- **问题**:
  - 候选池是"全体牧师法术牌"——这是一个上百张的固定全局池，对玩家几乎没有规划价值；
  - 没有动态条件（不限定法术学派、不限定费用、不限定职业/中立），悬停展示一个庞大无差别列表反而会污染界面；
  - 这是典型"看似可显示，实则没用"的随机生成牌。
- **修正建议**: 移除 extraDisplay。如确实要做，最多用一个轻量计数器（"已触发 X 次本局"），但收益极低。

### 6. 净场 (CORE_REV_252)
- **卡牌文本**: 消灭所有攻击力小于或等于3的随从。注能（）：改为小于或等于6。
- **当前建议 displayType**: `infuse_progress`
- **审核结论**: ✅ 保留
- **问题**:
  - 类别正确，注能是手牌实体级状态机；
  - 现有文案 `{progress}/{required}` 没有给出注能阈值，缺乏直观度（注能阈值是该牌专属：本局 4 个友方随从死亡）。
- **修正建议**:
  - `stateNeeded` 应写明"本牌进入手牌后友方随从死亡数 + 是否已 INFUSED"；
  - 悬停文案改为："注能 {current}/4（达成后改为消灭攻击力 ≤6 的随从）"；
  - 已注能时显示"已注能：消灭攻击力 ≤6 的随从"，progress 隐藏。

### 7. 灵魂向导 (CORE_AV_328)
- **卡牌文本**: 嘲讽，亡语：抽一张神圣法术牌和一张暗影法术牌。
- **当前建议 displayType**: `deck_target_highlight`
- **审核结论**: ⚠️ 需修正
- **问题**:
  - 亡语是从"牌库内的神圣 / 暗影法术"中抽，候选池就是玩家自己构筑里的内容；
  - 这种信息其实已经能从牌库列表的"按学派筛选"中看到，但若不做学派筛选 UI，则一个紧凑列表确实有帮助；
  - 当前文案把神圣 / 暗影合并成一段 `{cardNames}`，没有分组，容易让玩家误以为是同一池中抽两张。
- **修正建议**:
  - 拆为两个独立计数：`holySpellsInDeck`、`shadowSpellsInDeck`；
  - 悬停文案改为："牌库剩余 神圣法术 {holyCount} 张 / 暗影法术 {shadowCount} 张"，并分别列出名称；
  - 若任一池为 0，提示"无法抽到对应学派"。

### 8. 亡者卡特琳娜 (CORE_DAL_721)
- **卡牌文本**: 在你的回合结束时，复活另一个友方亡灵随从。
- **当前建议 displayType**: `graveyard_pool`
- **审核结论**: ✅ 保留
- **问题**:
  - 限定"亡灵"种族，但 `stateNeeded` 用了通用的 `eligibleDeathrattleOrMinionPool`，过于宽泛；
  - 触发位置是"回合结束时"，所以悬停最适合在它已上场时显示——`hand` 面板的展示价值远低于`board` 上的预览（候选会随回合改变）。
- **修正建议**:
  - `stateNeeded`：`friendlyUndeadGraveyardThisGame`；
  - `displaySurfaces` 加入 `board` 悬停（场上随从悬停时也显示候选）；
  - 文案：复活候选（友方亡灵，已死亡）：{cardNames}；共 {count} 张实例。
  - 注意：复活按"实例次数"加权（同一张死多次会被加权选中多次），所以这里展示的应是"实例数量"而非"去重种类"，与永恒奴役不同。

### 9. 小型法术钻石 (CORE_LOOT_507)
- **卡牌文本**: 复活两个不同的友方随从。（施放四个法术后升级。）
- **当前建议 displayType**: `graveyard_pool_and_upgrade_progress`
- **审核结论**: ⚠️ 需修正
- **问题**:
  - 这是一张可升级法术（4 个法术升级为下一阶），文案完全没体现升级进度；
  - "两个不同的"意味着是"种类去重"复活，与永恒奴役类似而与卡特琳娜不同；
  - `stateNeeded` 缺 `spellsCastSinceInHand`（或全局法术计数，取决于该牌实现）。
- **修正建议**:
  - `stateNeeded`：`friendlyDeadMinionDistinctPool`, `spellsCastProgressForUpgrade`；
  - 悬停文案：升级进度 {spellsCast}/4；复活候选（去重）：{cardNames}（共 {distinctCount} 种）；
  - 若已升级为更大版本（大型/巨型钻石），应只显示升级后的版本对应预览。

### 10. 永恒巨龙姆诺兹多 (CORE_DRG_090)
- **卡牌文本**: 战吼：使用你的对手上个回合使用的所有卡牌。
- **当前建议 displayType**: `last_turn_history`
- **审核结论**: ✅ 保留
- **问题**:
  - 上回合对手出牌信息玩家可以靠记忆，但漏看常见，悬停显示是高价值；
  - "上个回合"是动态的——每次轮到本回合开始时，对手的"上个回合"才确定；本回合结束后，"上个回合"会被更新；
  - `displaySurfaces` 应排除 `deck`（牌库里看上回合记录无意义）。
- **修正建议**:
  - `stateNeeded`：`opponentCardsPlayedPreviousTurn`；
  - 文案：对手上回合使用的卡牌：{cardNames}（共 {count} 张）。空则"对手上回合未使用卡牌"。
  - 注意"使用"包含法术 + 随从 + 武器，不含英雄技能。

### 12. 卡多雷精魂 (FIR_777)
- **卡牌文本**: 嘲讽。吸血。战吼：如果你在本回合中使用过英雄技能，获得+3/+3。
- **当前建议 displayType**: `condition_status`
- **审核结论**: ✅ 保留
- **问题**:
  - 条件简单（本回合是否用过英雄技能），但属于"无法从当前场面看出"的隐藏状态；
  - 文案 `{stateSummary}` 太抽象，未传达具体收益；
  - `stateNeeded` 写作 `matchHistoryStateForThisEffect` 也过于通用。
- **修正建议**:
  - `stateNeeded`：`heroPowerUsedThisTurn`；
  - 文案：本回合 {已/未} 使用英雄技能 → 战吼 {生效（+3/+3）/不生效}；
  - 推荐 implementationPriority: low（信息只在本回合有效、可视化简单，但价值不高）。

### 14. 艾维娜，艾露恩钦选者 (EDR_895)
- **卡牌文本**: 战吼：开启为期三回合的月相演变。当满月升起时，在本局对战中你的卡牌法力值消耗为（1）点。
- **当前建议 displayType**: `persistent_effect_timer`
- **审核结论**: ✅ 保留
- **问题**:
  - 出场后留下 3 回合倒计时这是典型"持续效果"——非常需要持久 UI；
  - 但她在手中时其实是"固定效果"——出场后才开始计时，所以"hand"悬停仅显示"出场后开启 3 回合月相"即可；
  - 在 `deck/graveyard/hand` 等位置全显示同一文案有冗余。
- **修正建议**:
  - 出场前（hand/deck/graveyard 悬停）：文案"打出后开启 3 回合月相，满月时本局卡牌费用变为 1"；
  - 出场后（board 悬停 + 全局徽章）：剩余 {n} 回合至满月；
  - 满月后（持久生效）：本局所有卡牌费用 = 1（这部分应该靠全局徽章/状态条提示，而非靠她单卡悬停）；
  - 强烈建议加全局月相 / 满月状态徽章（这才是真正的玩家价值）。

### 17. 林歌海妖 (TLC_819)
- **卡牌文本**: 吸血。如果你在本回合中使用过神圣和暗影法术牌，本牌的法力值消耗为（1）点。
- **当前建议 displayType**: `cost_condition_progress`
- **审核结论**: ✅ 保留
- **问题**:
  - 条件需同时满足"本回合用过神圣法术 AND 本回合用过暗影法术"；当前 `historyCounterForThisEffect` 只是单个计数器，不够；
  - 文案模糊，没说当前费用具体是几。
- **修正建议**:
  - `stateNeeded`：`holySpellPlayedThisTurn`, `shadowSpellPlayedThisTurn`, `currentEffectiveCost`；
  - 文案：本回合 神圣 {✓/✗}，暗影 {✓/✗}；当前费用 {1/6}；
  - 应在 hand 悬停 + hand 行内角标都显示当前实际费用（已经达成条件时为 1）。

### 18. 轮回转生 (TLC_818)
- **卡牌文本**: 复活法力值消耗为1，2，3的随从各一个，使其获得复生。
- **当前建议 displayType**: `graveyard_pool_by_cost`
- **审核结论**: ✅ 保留
- **问题**:
  - 候选必须按费用分桶（1 费 / 2 费 / 3 费）——当前文案 `{cardNames}` 不分桶；
  - 因为是"各一个"，每个桶之间是独立抽，所以应该分别展示；
  - 同一桶内是"实例加权"还是"去重"取决于游戏实现；目前实测炉石的"复活随机随从"系列基本是按实例数加权抽。
- **修正建议**:
  - `stateNeeded`：`friendlyDeadMinionsByCost{1,2,3}`；
  - 文案三段：1 费墓地：{names}（n 张实例）/ 2 费：…/ 3 费：…；
  - 任一桶为空时显著标红"1 费墓地无候选 → 该桶无随从复活"，这对决策最关键。

### 19. 擎天雷龙 (DINO_431)
- **卡牌文本**: 嘲讽。亡语：随机召唤一个法力值消耗大于或等于（5）点的嘲讽随从。
- **当前建议 displayType**: `candidate_pool`
- **审核结论**: ❌ 移除
- **问题**:
  - 候选池是"全体 ≥5 费嘲讽随从"，是固定全局池，列表庞大；
  - 与莱拉同理——展示一个无差别的大列表对玩家几乎没价值；
  - 没有动态变化（不依赖本局历史 / 牌库 / 墓地）。
- **修正建议**: 移除 extraDisplay。如保留，建议改成静态 tooltip"亡语：召唤 1 个 ≥5 费的嘲讽随从（从全体合法池中随机）"，但完全没必要单独占一个悬停槽。

### 21. 净化祭司 (CATA_216)
- **卡牌文本**: 战吼：在本局对战中，你的治疗效果恢复的生命值增加2点。
- **当前建议 displayType**: `persistent_effect_counter`
- **审核结论**: ⚠️ 需修正
- **问题**:
  - 效果是本局永久叠加（多张可叠加 +2/+2/…），需要展示"当前治疗加成总值"；
  - `affectedCards` 字段不太合适——这个效果不针对特定牌，而是所有治疗行为；
  - 建议把"本局已治疗总量"也加进来，对牧师玩家衡量持续收益很有用（属于牧师典型隐藏数值）；
  - hand / deck 悬停时，价值在于提示"打出后会叠加 +2"，已上场后则关心"当前总加成"。
- **修正建议**:
  - `stateNeeded`：`healingBonusActive`, `totalHealingBonusThisGame`, `purifierStacks`；
  - hand/deck 悬停文案：当前治疗加成 +{currentBonus}；本牌出场后将叠加为 +{currentBonus+2}；
  - 全局可考虑加"本局治疗总量"徽章（牧师专属，并非此卡专有，但此卡是最佳触发场景）；
  - 推荐 implementationPriority: medium。

## 总结建议

### 移除建议（2 张）
- **5. "太阳裂片"莱拉** — 候选池是"全体牧师法术"，固定且过大，无规划价值；
- **19. 擎天雷龙** — 候选池是"全体 ≥5 费嘲讽随从"，同样是固定大全集池。

> 通用准则：当随机候选池是一个**全局静态集合**（不被本局牌库 / 墓地 / 出牌历史筛选），悬停展示意义有限，应移除以减少视觉噪音。

### 高优先级实现（5 张，墓地池 / 注能 / 费用条件类）
- 3. 永恒奴役（去重墓地池）
- 8. 亡者卡特琳娜（亡灵墓地池 + 实例加权）
- 9. 小型法术钻石（升级进度 + 去重墓地池）
- 18. 轮回转生（按费用分桶墓地池，最重要：任一桶为空要醒目提示）
- 17. 林歌海妖（双学派条件 + 当前费用显示）

### 中优先级实现（4 张）
- 6. 净场（手牌注能进度 4 友方死亡）
- 7. 灵魂向导（牌库内神圣 / 暗影法术分别计数）
- 10. 永恒巨龙姆诺兹多（对手上回合出牌列表）
- 21. 净化祭司（本局治疗加成栈数，可顺带加治疗总量徽章）

### 低优先级实现（2 张）
- 12. 卡多雷精魂（仅本回合英雄技能是否使用，信息简单）
- 14. 艾维娜（建议靠全局月相状态徽章承载，而非靠她本人卡牌悬停）

### 牧师特性建议（跨卡通用，可作为后续提议）
1. **友方墓地池**追踪是牧师的核心——需要维护一个跨整局的死亡记录，并支持按"种类去重"、"实例加权"、"按费用分桶"、"按种族（亡灵 / 龙）筛选"四种视图；上述 4 张复活/发现牌可复用同一底层数据源。
2. **法术学派计数器**（神圣 / 暗影 / 自然 等）：本回合 / 本局两个粒度。林歌海妖、灵魂向导（虽是牌库筛选）、池中其他史诗甚至非候选卡都会受益。
3. **本局治疗总量 + 当前治疗加成**：牧师玩家关心数据，但游戏 UI 完全不显示；可独立做一个职业徽章，由净化祭司这类卡来触发显示。
4. **"上一回合 / 上一张 / 本回合"出牌历史**：姆诺兹多直接依赖，其它牌（如手牌中的"本回合已使用法术"型条件）也会用到，建议封装为通用 `playedHistoryWindow` 数据模型。
5. **嘲讽 / 亡语全集池**这类**静态全集随机**池一律不做悬停，符合"减少视觉噪音"原则——这条建议适用于所有职业，建议写入审核通用规则。
